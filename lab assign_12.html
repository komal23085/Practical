1. Promises and Async/Await

A Promise represents a value that may be available now, in the future, or never. It handles asynchronous operations like API calls.

How Async/Await Simplifies Code

While Promises use .then() chains, async/await allows us to write asynchronous code that looks like standard synchronous code, making it much easier to read and maintain.

// Using async/await for cleaner logic
async function getUserData() {
    try {
        const response = await fetch('https://api.example.com/user');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Error fetching data", error);
    }
}

2. Event Delegation & addEventListener

addEventListener() attaches an event handler to a specific element. However, attaching 100 listeners to 100 buttons is bad for performance.

Event Delegation

Instead of adding listeners to every child, we add one listener to the parent. Because of "Event Bubbling," clicks on children rise up to the parent.
